# Deploy — wire.phall.io

Hosting is **Cloudflare Pages** (matching the phall.io stack). `astro build`
emits a static `dist/`; a Pages Function (`functions/_middleware.ts`) adds
`Accept`-based content negotiation + CORS on top, and `public/_redirects`
(generated by `scripts/prebuild.ts`) handles the 301 guesses. No custom Worker,
no server at request time — just files plus a thin edge middleware.

## Prerequisites

- **Bun** (`brew install oven-sh/bun/bun`). Wrangler runs via `bunx`.
- A Cloudflare account; the `phall.io` zone already onboarded.
- One-time auth: `bunx wrangler login` (browser OAuth). CI uses the
  `CLOUDFLARE_API_TOKEN` + `CLOUDFLARE_ACCOUNT_ID` secrets instead.

## Build

```sh
bun install
bun run build      # = bun run scripts/prebuild.ts  &&  astro build
```

Output (`dist/`), all static:

```
dist/
  index.html  index.json                          home + root manifest
  terminal-osc/133.html  terminal-osc/133.json     page + canonical JSON twin (per entry)
  http-status.html  http-status/index.json         family landing + manifest (per family)
  search-index.json                                verified-entry search index (client-side)
  llms.txt  llms-full.txt                          agent index
  _redirects                                       301 table (/osc/133 → /terminal-osc/133, …)
  _astro/                                          hashed css/js
functions/_middleware.ts                           Accept negotiation + CORS (bundled by Pages)
```

## Deploy

```sh
bun run deploy     # = bun run build && bunx wrangler pages deploy ./dist --project-name=wire --branch=main
```

First deploy creates the `wire` Pages project. Then attach the domain (Cloudflare
dashboard → Workers & Pages → wire → Custom domains → add `wire.phall.io`).
Because `phall.io` is a Cloudflare zone, the CNAME and TLS cert are provisioned
automatically — no manual DNS.

## Verify the dual-serving

```sh
# locally (full Pages behavior incl. functions + _redirects):
bunx wrangler pages dev dist

curl -H 'accept: application/json' https://wire.phall.io/terminal-osc/133   # → JSON
curl https://wire.phall.io/terminal-osc/133                                 # → HTML
curl https://wire.phall.io/http-status/418.json                             # → JSON (suffix)
curl -so /dev/null -w '%{http_code} %{redirect_url}\n' https://wire.phall.io/osc/133   # → 301
curl https://wire.phall.io/llms.txt                                         # → text/plain
```

CI (`.github/workflows/deploy.yml`) runs `bun run gates` + `bun test` as hard
prerequisites, then builds and deploys on push to `main`.
